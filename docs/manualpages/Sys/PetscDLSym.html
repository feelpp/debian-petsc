PetscDLSym

 finds a symbol in a dynamic library

Synopsis

PetscErrorCode  PetscDLSym(PetscDLHandle handle,const char symbol[],void **value)
Not Collective

Input Parameters

handle - obtained with PetscDLOpen() or NULL
symbol - name of symbol

Output Parameter

value -pointer to the function, NULL if not found

Notes

If handle is NULL, the symbol is looked for in the main executable's dynamic symbol table. In order to be dynamically loadable, the symbol has to be exported as such. On many UNIX-like systems this requires platform-specific linker flags.
